------------------------------------------------------
PASOS PARA PROBAR CONSULTAS EN DOCKER
------------------------------------------------------

1. copiar la carpeta docker-compose-flink en un directorio 

2. acceder al directorio docker-compose-flink

3. tener en cuenta que en la configuración de docker-compose.yml, se especifica un
directorio para compartir datos entre el host y el contenedor en docker.
Este directorio está especificado como 
	volumes:
      		- ~/data:/data
donde ~/data es el directorio en el host y /data el directorio en docker
En el directorio ~/data copiar el dataset adjunto

3. ejecutar el comando: 
docker-compose up --scale taskmanager=n
donde n es el número de taskmanager

4. acceder a la interfaz web por el navegador a través de http://localhost/8081 si no se ha cambiado el puerto por defecto en el archivo docker-compose.yml

5. en la interfaz web, hacer clic en 
Submit new job
	Add new+
		adicionar el archivo query-1.0-SNAPSHOT.jar, adjunto en el correo
		presionar upload
	Seleccionar el archivo .jar cargado
En el campo Entry Class especificar la clase que se va a ejecutar, en ese caso 		corresponde a cada consulta, por ejemplo, para especificar la consulta 1, copiar 	lo siguiente: org.univalle.rdf.out.Query1a
En el campo Program Arguments especificar dos parámetros: el dataset y el directorio de salida de resultados. Ejemplo, --dataset /data/dataset.ttl --output /data/bsbm5/
Este directorio corresponde al dirección en docker y que a su vez es el compartido en el host.
Presionar el botón Submit

6. Repetir el paso 5 por cada consulta cambiando el nombre específico. NO hay necesidad de volver a subir el archivo .jar
